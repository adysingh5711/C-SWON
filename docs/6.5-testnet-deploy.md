# 6.5 Testnet Deployment Guide for C-SWON

Step-by-step instructions to deploy the C-SWON subnet on the Bittensor Testnet for staging and testing before moving to mainnet (`finney`).

## 1. Prerequisites

- **Bittensor SDK and CLI**: `pip install bittensor`
- **A provisioned Wallet**: coldkey + hotkey created
- **Test TAO**: Request from the [OpenTensor Discord](https://discord.com/channels/799672011265015819/1107738550373454028/threads/1331693251589312553)

## 2. Setting Up Wallets

```bash
# Owner wallet
btcli wallet new_coldkey --wallet.name owner

# Validator wallet
btcli wallet new_coldkey --wallet.name vali
btcli wallet new_hotkey --wallet.name vali --wallet.hotkey default

# Miner wallet
btcli wallet new_coldkey --wallet.name miner
btcli wallet new_hotkey --wallet.name miner --wallet.hotkey default
```

Ensure your `owner` wallet has Test TAO before proceeding.

## 3. Create the Subnet

```bash
btcli subnet create --network test --wallet.name owner
```

Note the `netuid` from the output. Verify:

```bash
btcli subnet list --network test
```

## 4. Register Validator and Miner

```bash
# Register Validator
btcli subnets register --network test --netuid <netuid> --wallet.name vali --wallet.hotkey default

# Register Miner
btcli subnets register --network test --netuid <netuid> --wallet.name miner --wallet.hotkey default

# Stake the Validator
btcli stake add --network test --wallet.name vali --wallet.hotkey default
```

## 5. Start the Subnet

```bash
btcli subnet start --netuid <netuid> --network test
```

## 5.1 Disable Commit-Reveal Weights

```bash
btcli sudo set --netuid <netuid> --param commit_reveal_weights_enabled --value false --network test --wallet.name owner
```

Verify:
```bash
btcli sudo get --netuid <netuid> --network test | grep commit_reveal
```

## 5.2 Configure Hyperparameters (Recommended)

```bash
# Lower tempo for faster iteration (default 360 → 60)
btcli sudo set --netuid <netuid> --param tempo --value 60 --network test --wallet.name owner

# Match weights_rate_limit to tempo
btcli sudo set --netuid <netuid> --param weights_set_rate_limit --value 60 --network test --wallet.name owner

# Lower immunity period
btcli sudo set --netuid <netuid> --param immunity_period --value 500 --network test --wallet.name owner

# For small miner count
btcli sudo set --netuid <netuid> --param min_allowed_weights --value 1 --network test --wallet.name owner
```

> **Weight submission cadence:** Weights submit every `max(tempo, weights_rate_limit)` blocks.

## 5.3 Enable Emissions (Root Subnet Registration)

```bash
btcli root register --subtensor.network test --wallet.name owner
btcli root weights --subtensor.network test --wallet.name owner
```

## 6. Run C-SWON

> **CLI flag conventions:**
> - `btcli` commands use `--network test`
> - Neuron scripts use `--subtensor.network test`
> - These are NOT interchangeable

### Required Environment Variables

```bash
export CSWON_MOCK_EXEC=true
export CSWON_SYNTHETIC_SALT=$(python -c "import secrets; print(secrets.token_hex(32))")
```

> **CRITICAL:** Both variables are required. Save the salt value for reuse across restarts.

### Run the Miner

```bash
python neurons/miner.py \
  --netuid <netuid> \
  --subtensor.network test \
  --wallet.name miner \
  --wallet.hotkey default \
  --wandb.off
```

### Run the Validator

```bash
python neurons/validator.py \
  --netuid <netuid> \
  --subtensor.network test \
  --wallet.name vali \
  --wallet.hotkey default \
  --wandb.off
```

## Potential Issues

1. **WandB**: Use `--wandb.off` to disable logging (recommended for testnet).
2. **`--netuid` is required**: No default netuid — must always pass it.
3. **Environment variables**: Ensure `CSWON_MOCK_EXEC` and `CSWON_SYNTHETIC_SALT` are set before running.
