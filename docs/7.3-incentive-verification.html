Miner DAG construction is task-specific, not hardcoded. neurons/miner.py._design_workflow() reads the task description and available_tools from the synapse and:

  1. Calls _infer_required_capabilities(desc) — keyword/phrase matching against the description to determine the ordered action chain (e.g. generate_code → generate_tests for a "write and test" task)
  2. Calls _pick_subnet_by_capability() — ranks allowed subnets by capability type match and then by (avg_cost, avg_latency) from the SubnetProfiler historical observations
  3. Inserts ${prev_id.output.text} DataRef references between sequential nodes
  4. Caps at 10 nodes; falls back to a single-step pipeline only if no capability signals match

The SubnetProfiler accumulates real cost/latency observations per subnet across queries, so routing decisions improve over time. This is the miner's core intelligence — not a static template.

Proof 3: Incentive Mechanisms Behave as Intended​

Success-First Gating Works​

Miner UID X:
  S_success   = 0.650 (below 0.70)
  S_cost      = 0.000 (GATED)
  S_latency   = 0.000 (GATED)
  S_reliability = 0.800
  S = 0.50×0.650 + 0.25×0.000 + 0.15×0.000 + 0.10×0.800 = 0.405

  Without gating: S = 0.708 → Gating reduced score by 43%

VRF Anti-Gaming Works​
