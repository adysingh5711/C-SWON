# 6.2 Running on Mainnet

This guide covers deploying C-SWON on the Bittensor mainnet (`finney`).

**IMPORTANT:** Before mainnet deployment:
- First run [Running Locally](6.3-running-on-staging.md)
- Then run [Running on Testnet](6.1-running-on-testnet.md)

Your incentive mechanisms on mainnet are open to anyone and emit real TAO.

:::danger Security
- Do not expose your private keys.
- Do not reuse testnet wallet passwords.
- Ensure your incentive mechanism is resistant to abuse.
:::

## Prerequisites

- [Bittensor SDK](https://github.com/opentensor/bittensor#install) installed
- Sufficient TAO for subnet creation (check lock cost below)

## 1. Install C-SWON

```bash
git clone https://github.com/adysingh5711/C-SWON.git
cd C-SWON
python -m pip install -e .
```

## 2. Create Wallets

```bash
btcli wallet new_coldkey --wallet.name owner
btcli wallet new_coldkey --wallet.name miner
btcli wallet new_hotkey --wallet.name miner --wallet.hotkey default
btcli wallet new_coldkey --wallet.name validator
btcli wallet new_hotkey --wallet.name validator --wallet.hotkey default
```

## 3. Check Subnet Lock Cost

```bash
btcli subnet lock_cost
```

By default you must have at least 100 TAO. The exact amount fluctuates with demand.

## 4. Create Subnet

```bash
btcli subnet create --wallet.name owner
```

Note the `netuid` assigned.

## 5. Register Keys

```bash
# Register Miner
btcli subnet recycle_register --netuid <netuid> --subtensor.network finney \
  --wallet.name miner --wallet.hotkey default

# Register Validator
btcli subnet recycle_register --netuid <netuid> --subtensor.network finney \
  --wallet.name validator --wallet.hotkey default
```

## 6. Verify Registration

```bash
btcli wallet overview --wallet.name validator
btcli wallet overview --wallet.name miner
```

## 7. Configure Environment

```bash
# For mainnet with live execution:
export CSWON_MOCK_EXEC=false
export CSWON_PARTNER_HOTKEY=<your_registered_hotkey_on_partner_subnets>
export CSWON_SYNTHETIC_SALT=<your-persistent-salt>
```

## 8. Run Miner and Validator

```bash
# Miner
python neurons/miner.py --netuid <netuid> --wallet.name miner --wallet.hotkey default

# Validator
python neurons/validator.py --netuid <netuid> --wallet.name validator --wallet.hotkey default
```

## 9. Get Emissions Flowing

```bash
btcli root register --wallet.name owner
btcli root weights --wallet.name owner
```

## 10. Monitoring

After the first tempo (~72 minutes with default tempo=360):
- Check `vtrust` in the metagraph — should begin climbing after tempo 3-5
- Monitor incentive and emission columns
- Use the C-SWON dashboard for real-time scoring data

## Security Considerations

- Use separate wallets for owner, validator, and miner roles
- Keep coldkeys on secure, isolated machines
- Monitor for unusual scoring patterns
- Keep the codebase updated with latest security patches
