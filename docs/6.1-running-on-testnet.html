6.1 Running on Testnet

This guide describes how to create and run the C-SWON subnet on the Bittensor test network. We recommend you first try Running Locally before attempting testnet deployment.

Security
  • Do not expose your private keys.
  • Only use your testnet wallet for testing.
  • Do not reuse the password of your mainnet wallet.
  • Make sure your incentive mechanism is resistant to abuse.

Considerations​

Research Existing Subnets​

  • Real-time subnet info on tao.app
  • LearnBittensor's Subnet listings

Burn Cost​

The burn cost is dynamic: it reflects current registration demand, lowers gradually over time, and doubles every time a new subnet is created.

Subnet Creation Rate Limits​

Limited to one subnet creation per 28800 blocks (~4 days).

Prerequisites​

  1. Python 3.10+ installed
  2. Install BTCLI: 
    pip install bittensor-cli
  3. Install C-SWON: 
    git clone https://github.com/adysingh5711/C-SWON.git
    cd C-SWON
    python -m pip install -e .
  4. Create Wallets: 
    btcli wallet create --wallet.name owner --wallet.hotkey default
    btcli wallet create --wallet.name miner --wallet.hotkey default
    btcli wallet create --wallet.name validator --wallet.hotkey default
  5. Get Testnet TAO: Request from the Bittensor Discord.

1. Check the Burn Cost​

btcli subnet burn-cost --network test

2. Create the Subnet​

btcli subnet create --network test --wallet.name owner

Note the netuid from the output.

3. Start the Subnet​

btcli subnet start --netuid <your-netuid> --network test

4. Disable Commit-Reveal Weights​

Testnet defaults to commit_reveal_weights_enabled=true, which causes signature errors on weight submission:

btcli sudo set --netuid <netuid> --param commit_reveal_weights_enabled --value false --network test --wallet.name owner

5. Configure Hyperparameters (Recommended)​

# Lower tempo for faster iteration
btcli sudo set --netuid <netuid> --param tempo --value 60 --network test --wallet.name owner

# Match weights_rate_limit to tempo
btcli sudo set --netuid <netuid> --param weights_set_rate_limit --value 60 --network test --wallet.name owner

# Lower min_allowed_weights for small miner count
btcli sudo set --netuid <netuid> --param min_allowed_weights --value 1 --network test --wallet.name owner

6. Register Keys​

# Register Miner
btcli subnet register --netuid <netuid> --network test --wallet.name miner --wallet.hotkey default

# Register Validator
btcli subnet register --netuid <netuid> --network test --wallet.name validator --wallet.hotkey default

# Stake the Validator
btcli stake add --network test --wallet.name validator --wallet.hotkey default

7. Set Environment Variables​

export CSWON_MOCK_EXEC=true
export CSWON_SYNTHETIC_SALT=$(python -c "import secrets; print(secrets.token_hex(32))")

CRITICAL: Both variables are required. Save the salt value for reuse across restarts.

8. Run Miner and Validator​

CLI flag note: btcli uses --network test, neuron scripts use --subtensor.network test. These are different CLI frameworks.

# Run Miner
python neurons/miner.py \
  --netuid <netuid> \
  --subtensor.network test \
  --wallet.name miner \
  --wallet.hotkey default \
  --wandb.off

# Run Validator (separate terminal)
python neurons/validator.py \
  --netuid <netuid> \
  --subtensor.network test \
  --wallet.name validator \
  --wallet.hotkey default \
  --wandb.off

9. Get Emissions Flowing​

btcli root register --subtensor.network test --wallet.name owner
btcli root weights --subtensor.network test --wallet.name owner

10. Verify​

btcli wallet overview --wallet.name validator --network test
btcli wallet overview --wallet.name miner --network test
